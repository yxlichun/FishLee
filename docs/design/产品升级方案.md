# AI学习路径产品升级建议

> 从"AI PM学习路径"（单用户、单路径、无AI）升级为通用的AI辅助学习路径产品

---
## 项目名称
安石

## 数据层次抽象
1. 当前是面向单个目标的学习路径，已经不够了，我希望这个站点能够面向多个学习/生活目标进行路径拆解和进度跟踪，所以首先要抽象一层；

2. 数据结构改进，引入一个goal的实体，学习路径、计划、笔记、资源、打卡等都挂在这个实体之下；

3. 页面上来讲，一级页面先只显示几个目标，每个目标点入进去就是当前的目录结构；


4. 我希望在goals页面增加一个可爱的小鱼的悬浮按钮，当点击这个按钮时弹出一个面板，此面板对接了大模型，会根据每日的登录，打卡，预期的计划，进展等信息进行总结，基本的结构是，①打招呼；②今日计划是什么，你当前的学习进度是什么；③今日完成的总结（打卡，编制计划，撰写文档，灵感等）。所包含的信息均为多个目标的信息。模型我计划使用火山引擎提供的豆包大模型，已经做了申请。
